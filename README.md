# PIZZA_CLUD--PIZZA_SALES_ANALYSIS--SQL
A comprehensive SQL-based analysis of pizza sales data to track revenue, customer trends, and operational metrics. It features a relational database schema and optimized MySQL queries designed to turn raw transactional data into actionable business insights.


This project focuses on analyzing transactional data for a pizza restaurant to uncover key business insights. By leveraging MySQL, I performed data modeling and executed complex queries to evaluate revenue performance, customer behavior, and operational efficiency.

Key Features & Insights:

Revenue Tracking: Calculated total sales and identified the top 3 highest-grossing pizza types to understand profitability.

Order Patterns: Analyzed peak ordering hours and the most common pizza sizes to optimize kitchen operations.

Product Performance: Evaluated sales volume by category and identified the top 5 most popular pizzas.

Operational Metrics: Calculated the average number of pizzas ordered per day to assist in inventory management.

Database Schema:

The project uses a relational structure consisting of the following tables:

orders: Contains order IDs, dates, and timestamps.

orders_details: Maps individual pizzas and quantities to specific orders.

pizzas: Stores pricing and size details.

pizza_types: Contains categorization and naming for each product.
